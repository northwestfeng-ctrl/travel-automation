# 酒店资料目录

自动客服回复会优先按以下顺序读取酒店资料：

1. `auto_reply_accounts.local.json` 里的 `hotel_info_file`
2. `hotel_info/<eBooking酒店ID>.md`
3. `hotel_info/<携程酒店ID>.md`
4. `hotel_info/<account_id>.md`

如果上面都没有命中，脚本还会继续尝试自动读取：

1. 当前账号页面上下文里的主酒店信息
2. 公开携程详情页里的基础酒店资料

这样即使没有先手工建 Markdown，地址、停车、Wi-Fi 一类基础问答也能按对应酒店回复，而不是直接串用别家资料。

建议每个酒店一份独立 Markdown，至少补齐这些信息：

- 名称
- 地址
- 入住时间
- 退房时间
- 早餐
- 宠物政策
- 停车
- Wi-Fi
- 泳池/餐厅等高频问答项

如果自动读取也失败了，客服脚本才会回退到保守的“需要确认后回复”话术，而不会继续套用其他酒店的资料。
